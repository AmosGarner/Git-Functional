Terminology: 1/2

Functor
Set of rules that are expected from a function's I/O.
Lambda
A function that can be treated like a value.
Monaid
A function that takes in a input type and returns that same type.
Monad
An object containing a return and a chain.
* Contracts are how functions handle error reporting. * You can think of these like callback functions. * Addition is a monaid because it takes in 2 ints and returns the sum of those ints as an int. * A chain function is like the map function. So a monad would loop through some data and return something.

Terminology: 2/2

Mutability
Determines if data can or cannot change.
Side Effects
Code in this function interacts with an external muttable source.
Purity
The return value must only rely on its inputs without side-effects.
Currying
Converting a function with multiple arguments into a one-at-a-time function.
